# 代码审查清单

## 必查项 (Blocker)

### 安全
- [ ] 无硬编码密钥/密码
- [ ] 用户输入已做校验和转义
- [ ] SQL 使用参数化查询
- [ ] 敏感数据已脱敏（手机号、身份证等）
- [ ] 高风险操作有风控检查

### 功能
- [ ] 功能符合需求描述
- [ ] 边界情况已处理
- [ ] 错误处理完整（不吞异常）
- [ ] 日志记录关键节点（含 request_id）

### 测试
- [ ] 有单元测试覆盖核心逻辑
- [ ] 测试用例覆盖正常/异常路径
- [ ] Mock 不过度，真实依赖要测试

---

## 应查项 (Major)

### 代码质量
- [ ] 函数职责单一（≤ 50 行）
- [ ] 无重复代码（DRY）
- [ ] 命名清晰，无缩写
- [ ] 无魔法数字/字符串

### 性能
- [ ] 无 N+1 查询
- [ ] 热路径有缓存
- [ ] 批量操作正确

### 可观测性
- [ ] 关键操作有 metrics
- [ ] 异常有追踪（trace_id）
- [ ] 日志级别正确（INFO/WARN/ERROR）

---

## 建议项 (Minor)

- [ ] 注释说明 WHY 而非 WHAT
- [ ] 复杂逻辑有简要说明
- [ ] TODO 有关联 Issue

---

## 审查用语

**通过**: "LGTM" / "✅ 看起来不错"

**需修改**: 
- "建议改为..."（Minor）
- "需要修改：..."（Major）
- "必须修改：..."（Blocker）

---
**How to apply**: Review 时逐项检查，Blocker 不通过则拒绝合并。
